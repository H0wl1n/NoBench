# 🎮 NoBench (TeamFinder)

> Сервис для поиска тиммейтов по командным видеоиграм: создаёшь профиль, публикуешь объявление о поиске напарника — и находишь команду по игре, рангу и роли.

**Ценность одним предложением:** NoBench за 30 секунд помогает игроку найти напарника с подходящим рангом и ролью — вместо поиска через сторонние чаты и форумы.

## ✨ Возможности (страницы)

| Страница | Что внутри |
|---|---|
| **Главная** | Лента последних объявлений о поиске тиммейтов |
| **Объявления** | Список объявлений с фильтром по игре / рангу / роли |
| **Объявление** | Подробная карточка объявления + профиль автора |
| **Создать объявление** | Форма публикации: игра, нужная роль, ранг, описание |
| **Игроки** | Список зарегистрированных пользователей с количеством лайков |
| **Профиль игрока** | Никнейм, возраст, игра, ранг, роль, описание, объявления |
| **Создать профиль** | Регистрация: никнейм, возраст, игра, ранг, роль |
| **Сообщения** | Список диалогов и переписка с конкретным игроком |

## 🛠️ Технологический стек

| Слой | Технология | Почему |
|---|---|---|
| **Frontend** | React 18 + Vite | Компонентный подход, быстрый dev-сервер с HMR |
| **Роутинг** | react-router-dom | Переходы между страницами без перезагрузки (SPA) |
| **Стили** | Bootstrap 5 + Bootstrap Icons | Готовая система компонентов, не тратим время на вёрстку с нуля |
| **Backend** | Node.js + Express | Один язык с фронтом, минималистичный фреймворк, низкий порог входа |
| **БД** | SQLite (`node:sqlite`) | Встроена в Node.js, не нужен отдельный сервер БД — легко запускать и тестировать |
| **Связь front/back** | REST API (JSON) + прокси Vite (`/api`) | Фронт и бэк общаются по HTTP, в dev-режиме CORS не мешает за счёт proxy |

> Бэкенд поднимается на `http://localhost:3001`, фронтенд — на `http://localhost:3000`. Запросы с фронта на `/api` автоматически идут на бэкенд (настроено через proxy в `vite.config.js`), поэтому отдельно указывать адрес бэкенда не нужно.

## 🚀 Как запустить

1. Установить **Node.js LTS** (версии 22+) с [nodejs.org](https://nodejs.org), если ещё не установлен.

2. Клонировать репозиторий:
   ```bash
   git clone https://github.com/H0wl1n/teamfinder.git
   cd teamfinder
   ```

3. Установить и запустить backend:
   ```bash
   cd backend
   npm install
   npm run dev
   ```
   Сервер поднимется на [http://localhost:3001](http://localhost:3001) (база данных `teamfinder.db` создастся автоматически при первом запуске, с тестовыми данными).

4. В отдельном терминале установить и запустить frontend:
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
   Приложение откроется на [http://localhost:3000](http://localhost:3000).

5. Открыть [http://localhost:3000](http://localhost:3000) в браузере. Запросы с фронтенда на `/api` автоматически перенаправляются на backend (настроено через proxy в `vite.config.js`), отдельно указывать адрес бэкенда не нужно.
